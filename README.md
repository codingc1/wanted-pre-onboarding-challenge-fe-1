### 프로젝트 회고

* 그동안 apollo-graphQL만 써왔던 터라 라이브러리를 바꿔써야하는 이번 코스에 참여할때 적응할 수 있을지 부담이 되었으나 과제를 하나씩 수행하면서 해결할 수 있어 좋았다.
 - react-query, reat-route v6, redux 가 비교적 최근 것들이고 변동이 있어 참고 자료가 적어 어려움을 겪었는데 그 덕에 공식문서를 몇번이고 보게 되고 적용하면서 경험도 되고 공부도 되었다.

* 다른 동기의 코드를 많이 보았다. 다른 사람 코드를 보면 배끼는것 같고 내가 만들면 되지라는 생각을 했는데 동기 코드의 장점이 같은 라이브러리를 쓰니 눈에 쏙쏙 들어와서 짧은 시간에 죽 둘러볼 수 있어 좋았다. github예제들을 여러개 둘러보는것도 예전에는 열기만해도 어지러웠는데 자꾸 뒤적뒤적하니 한번 봐야지라는 용기가 생겼다. 다른 사람 코드를 많이 봐야겠다고 이번 코스에서 생각을 바꾸게 되어 좋았다.

* 각 라이브러리 공식문서를 더 많이 보게되었다. 
  - reat-query를 동기 코드를 보고 적용하지만 결국은 내 스타일대로 만들려면 공식문서를 보게 되었다. 특히 redux를 설치할때 굉장히 쉽게 생각했는데 어려움을 겪어 놀랬다. 예제들도 다 스타일과 라이브러리가 다르고, 4달 전 예제도 안되고 공식문서는 이전 사용자 중심으로 보여주다보니 그대로 붙여넣기도 안되서, 열심히 다른 것들도 검색해서 해결했는데 스택오버플로에 나처럼 공식문서 보고 오류뜬 질문이 올라와서 웃음이 나기도 했다. 그래도 라이브러리를 무사히 내 프로젝트에 적용하여 뿌듯했다.

  - react-query : graphQL과 비슷하여 너무 마음에 들고 재미있었다. 
  - route v6 : protect Route를 만들어서 그 안에 props로 element를 넣어 한 줄로 구현하려고 했으나 실패하고 공식문서대로 하고 말았다ㅠㅠ 이전 버전에서는 잘 되는거 같았는데 새 버전에서는 안되는듯하다.
  - redux-toolkit : 부트캠프에서 썼을때는 너무나도 코드량이 많아서 왠만하면 안써야겠다 할 정도였는데 이번에 사용해보니 정말 편리해져서 신선했다.
